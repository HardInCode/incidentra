## 2. Indexing

IEEE Access is indexed in Scopus, Web of Science (Science Citation Index Expanded), EI Compendex, IET Inspec, DOAJ, and Google Scholar. As of 2024, the journal has an Impact Factor of 3.6 and a CiteScore of 9.0.

## 3. Scope of the Journal

IEEE Access covers all IEEE fields of interest, including computer science, information security, networking, and systems engineering. It accepts both applied and theoretical work, though the journal is particularly well-known for publishing practical system implementations. Topics like intrusion detection, web application security, SOC automation, and AI-assisted threat response are regularly featured.

## 4. Relation to Capstone Project

Incidentra is a web-based SOC platform that handles log ingestion, attack detection using OWASP-based regex patterns, automated IP blocking, and AI-generated incident explanations via Groq/LLaMA. The AI role in Incidentra is strictly for analysis — generating human-readable threat summaries, reasoning, recommended actions, and MITRE ATT&CK mappings per incident — not for automation. These topics align well with what IEEE Access publishes: practical security system implementations combining intrusion detection, log-based threat analysis, and LLM applications in cybersecurity.

## 5. Example Article

**Title:** *Application of Large Language Models in Cybersecurity: A Systematic Literature Review*

**Authors:** Hasanov, I., Virtanen, S., Hakkala, A., & Isoaho, J. (2024). *IEEE Access*, Vol. 12, pp. 176751–176778. DOI: 10.1109/ACCESS.2024.3505983

This systematic literature review surveys how LLMs are applied across cybersecurity domains, including threat detection, log analysis, alert explanation, and incident response support. Critically, the paper identifies "providing natural language explanations for security alerts" and "contextual reasoning for intrusion detection" as key LLM use cases — precisely what Incidentra implements using LLaMA via Groq API to generate human-readable summaries, threat reasoning, recommended actions, and MITRE ATT&CK mappings for every detected incident. This makes it a direct and strong reference for Incidentra's AI analysis component.
