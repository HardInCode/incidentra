- ☐ **BELUM LULUS** — Bloker: _______________

**Bloker umum & solusi cepat:**


| Gejala                             | Solusi                                                         |
| ---------------------------------- | -------------------------------------------------------------- |
| Insiden tidak muncul               | `USE_SIMULATED_LOGS=false`, path log benar, backend jalan      |
| Semua request jadi SCANNER di Burp | Ganti User-Agent ke browser normal                             |
| Docker backend restart             | `docker compose logs backend` — pastikan `DATABASE_URL` pakai `postgresql+psycopg://`; tunggu Postgres |
| UI tidak panggil API               | Manual: cek port 5000; Docker: rebuild frontend jika ganti URL |
| 403 terus di vuln-web              | Unblock di Blocked IPs atau `reset_smeguard.py`                |
| Docker build: `ajv/dist/compile/codegen` | `docker compose build --no-cache frontend` (setelah update Dockerfile) |

### Catatan pengujian (B3, B5)

**B3 — PATH_TRAVERSAL vs LFI_RFI:** Engine memilih **PATH_TRAVERSAL** jika payload `?file=../../` tanpa `php://` / URL remote (tie-break di `detection_engine.py`). Demo C7 (`php://filter/...`) tetap **LFI_RFI**.

**B5 — Rate limit vs Blocked IP:** Kelola di SOC **IP Management** → tab **Rate Limited** (`/api/rate-limited/`): lihat daftar, **Clear**, **Extend** TTL Redis + sinkron `rate_limited.json`. Unblock di tab Blocked juga menghapus rate limit.
